Project Overview INTERNAL DGTL Group Holdings Ltd.

Platform
Rebuild & Dashboard

A full migration of dgtlgroup.io from WordPress to Next.js 15 + Payload CMS v3, paired with the greenfield build of a client dashboard at dashboard.dgtlgroup.io — giving DGTL's clients a single authenticated portal for files, invoices, tasks, announcements, and work requests. Both products run as a co-located monorepo on Hostinger, powered by a self-hosted PostgreSQL on the VPS and Cloudinary for media. Frappe ERPNext serves as the financial and project backbone.

Dashboard Access Roles

Four roles govern the dashboard. Will is the Owner — a protected role that cannot be removed, demoted, or deleted by anyone, including other admins. Other users can be promoted to Admin by the Owner or any Admin, giving full platform access without the ability to touch the Owner account.

Phase 05
Dashboard v1 — Core Build
⬡ Dashboard

The client dashboard is built inside the monorepo at dashboard.dgtlgroup.io. Payload's built-in JWT auth is used — email/password login for all users. Four roles are implemented: Owner (Will — hardcoded, protected, cannot be removed or demoted by any action), Admin (can be assigned by Owner or other Admins, full platform access), Team Member, and Client. Role enforcement is applied at both the middleware and Payload collection access-control level — an Admin cannot demote or delete the Owner account. All v1 modules are built: file library, invoice view, task list, announcements, work request form, and the client info panel. Each module can be independently toggled per client from the Payload admin. The work request form routes submissions to Frappe ERPNext via the REST API and triggers a Dijital (Hermes Agent) WhatsApp summary to Will via n8n.

Phase 06
ERPNext Integration & API Layer
⚙ Integration

Frappe ERPNext is deployed on the VPS (Docker) and integrated with the dashboard via its REST API. API keys are provisioned per role — the dashboard reads invoices and tasks as a read-only service account. Frappe webhooks fire to n8n on invoice creation, task updates, and project status changes — triggering dashboard cache invalidation and optional WhatsApp alerts via Dijital. The work request form creates a new Frappe task/project via the API. Claude Code handles API discovery and integration testing iteratively.

Deploy Frappe ERPNext on Docker VPS — configure company and accounts
Provision Frappe API keys — read-only dashboard service account
